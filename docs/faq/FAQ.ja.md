# FAQ

**1. シークレットはどこに保存されますか？**

アプリケーションメモリ（RAM）に暗号化されて保存されます。

**2. シークレットはどのように保護されますか？**

データはクライアント側（ブラウザ）でAES 256、32バイトキーで暗号化されます。その後、シークレットへの特別なURLを受け取ります。
サーバー側は元のメッセージについて何も知りません。暗号化されたデータのみを（RAMに）保存します。

シークレットURLを開くと、クライアント（ブラウザ）がサーバー側から暗号化されたデータを読み込みます。その後、クライアント側は
URLにエンコードされた強力なキーでデータを復号化します。

シークレットURLには有効期限があります（最大1時間、2時間、または1日）。

**3. 暗号化にはどのアルゴリズムが使用されますか？**

AES 256、キー長32。

**4. メッセンジャー経由で1回限りのリンクを送信できますか？**

はい、リンクは期限切れになりません。

**5. アプリケーションに必要なリソースは？**
